  2. In Unity, open the New Atlantis project from the Git repo.



  3. Unity will open the project to an empty scene.



  4. In the Assets > NewAtlantis > Core > Scenes folder open the NewAtlantisViewer3.unity. scene.



  5. At this point you can run the project to launch the NewAtlantis viewer. If you have an existing login and password use that to connect to the NA Server. Otherwise create a unique login and password.



  6. To develop your own content, create a working directory within Assets. In the below example I created folder Assets > Rob. Within that folder I created a new Scene and then launched that blank scene. Within that scene I created a simple object consisting of 2 cubes and a sphere. To package them as a prefab, I created a blank GameObject, renamed it to "myPrefab", and then made my cube-sphere-cube construct a child of that GameObject by dragging them into "myPrefab" in the left menu. To create a prefab, I simply then dragged "myPrefab" down into my "Rob" folder in the Project viewer.



  7. To export this prefab into a form compatible with the New Atlantis architecture, right-click on the prefab in the Project viewer and select the option "Create a New Atlantis Asset Bundle from your prefab". Save the bundle into the NewAtlantisUnity > Bundles folder.



  8. To upload this prefab to the New Atlantis servers, load the launch the NewAtlantisViewer3.unity scene, and login to your user account. Select the MyNA button from the top-left menu. Within the MyNA window, click the Import a new asset to my library.



  9. From the dialog menu that pops up, select your prefab (for me its "robs_awesome_prefab.unity3d"). In the "name" field, give your file a name so that we can identify it. For this course, give your file a prefix "dmg_" for "Designing Musical Games". Make sure you leave the "public" button checked so that others can download your prefab.



  10. Once the file has been successfully uploaded, it will appear in your assets list and can be accessed in New Atlantis.



  11. If your project is using custom scripts, those scripts must be added to the New Atlantis Git repository. They need to be checked by Jonathan and/or Ben before they will work with the server. Please email us your scripts so they can be checked.
